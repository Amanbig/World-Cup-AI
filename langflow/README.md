# MatchMind — Langflow Setup

This directory contains the Langflow flows and helper scripts to integrate Langflow as the orchestration layer for MatchMind.

Two ways to get the flow into Langflow: **script** (automated) or **manual UI import**.

---

## 🚀 Option A — Script (automated)

```bash
cd langflow/
python3 create_flow.py
```

The script builds the flow JSON, saves it as `matchmind_flow.json`, and uploads it to your running Langflow instance.

**Requirements in `langflow/.env`:**
```env
LANGFLOW_API_KEY=sk-...        # from Langflow UI → Settings → API Keys
LANGFLOW_URL=http://localhost:7860   # optional, this is the default
OPENAI_API_KEY=sk-...          # needed by the LLM node at runtime
```

On success, it prints the `LANGFLOW_FLOW_ID` — copy that into your root `.env`.

---

## 🎨 Option B — Build the flow by hand in the UI

First, start Langflow:
```bash
# Docker
docker compose up langflow

# or local install
langflow run
```
Open [http://localhost:7860](http://localhost:7860) in your browser.

---

### Option B1: Simple RAG Chain (Fast & Deterministic)

Lays out a standard prompt-based retrieval chain:
```
[ChatInput] → [Chroma] → [Prompt] → [LLM Model] → [ChatOutput]
```

1. **User Question**: Drag a `Chat Input` node onto the canvas.
2. **Chroma DB**: Drag `Chroma` onto the canvas:
   - **Collection Name**: `fifa_rules`
   - **Persist Directory**: `/chroma_db` (if running inside Docker) or `./server/chroma_db` (local).
   - **Embedding Model**: `all-MiniLM-L6-v2`
3. **Prompt**: Drag a `Prompt` node and paste the template (located in `langflow/matchmind_flow.json` or generated by `create_flow.py`). Make sure to instruct the model to output the JSON visualization coordinates.
4. **LLM**: Drag a model provider (like `OpenRouter`, `Groq`, or `OpenAI`) and input your API keys.
5. **Chat Output**: Drag a `Chat Output` node.
6. **Connections**:
   - `ChatInput.message` ➔ `Chroma.search_query` & `Prompt.question`
   - `Chroma.results` ➔ `Prompt.context`
   - `Prompt.prompt` ➔ `LLM.input_value`
   - `LLM.text` ➔ `ChatOutput.input_value`

### Option B2: Agentic Tool-Calling Flow (Recommended)

Lays out an agentic loop utilizing the three custom backend tools:
```
[FifaRulesTool] ──┐
[WebSearchTool]  ──┼➔ [Tool Calling Agent] ➔ [ChatOutput]
[PitchAnimTool] ──┘         ▲
                            │
                      [LLM Provider]
```

#### 1. Setup the Custom Tools
Drag three **Custom Component** nodes onto the canvas. Open their code editors and paste the corresponding code snippets from `langflow/custom_components.py`:

*   **FIFA Rules Search Tool**: Queries the Chroma DB collection dynamically.
*   **Web Search Tool**: Uses `duckduckgo-search` to find news and recent match info.
*   **Pitch Animation Tool**: Allows the agent to draw tactical formations, offside plays, and VAR reviews.

#### 2. Setup the Agent and LLM
1.  **Tool Calling Agent**: Search and drag the `Tool Calling Agent` node onto the canvas.
2.  **LLM Provider**: Drag a model provider (e.g. `OpenRouter`, `Groq Model`, or `OpenAI Model`) onto the canvas. Do NOT use the generic `Language Model` executor.
3.  **Chat Input/Output**: Drag the standard `Chat Input` and `Chat Output` nodes.

#### 3. Connect the Nodes
*   **Tools**: Connect the `Tool` output handle of **all three** Custom Tool components directly to the single **Tools** input handle of the `Tool Calling Agent` (it supports multiple incoming connections).
*   **LLM**: Connect the model provider's output to the **Language Model** input port of the `Tool Calling Agent`.
*   **Inputs/Outputs**:
    - Connect `Chat Input` to the **Input** port of the `Tool Calling Agent`.
    - Connect the `Response` output of the `Tool Calling Agent` to the `Chat Output` node.

## 💾 Saving the Flow ID

After saving your flow in the UI, grab the Flow ID (the UUID in your browser URL):  
`http://localhost:7860/flow/<FLOW_ID>`

Add it to your root `.env`:
```env
LANGFLOW_URL=http://localhost:7860
LANGFLOW_FLOW_ID=<paste-uuid-here>
```

Then restart the services:
```bash
docker compose restart app
```
